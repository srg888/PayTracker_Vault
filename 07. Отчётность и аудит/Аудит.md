## Назначение
Неизменяемый журнал всех значимых действий в системе. Обязателен для казначейского процесса — при работе с финансовыми данными и 50-100 пользователями должна быть возможность восстановить, кто и когда что сделал.

## Что логируется
- Смена статуса заявки: кто, когда, из какого статуса в какой, комментарий (если был) — см. [[Статусы заявки]].
- Изменение любого поля заявки после подачи (было значение / стало значение) — см. [[Бизнес-правила|BR-012]].
- Загрузка / удаление файла (кто, когда, тип документа) — см. [[Справочник типов документов]].
- Назначение / переназначение Исполнителя — см. [[03. Назначение исполнителя|Назначение исполнителя]].
- Отклонение заявки (кто, причина).
- Действие Заказчика "Ознакомлен с причиной отклонения" (кто, когда) — неотменяемое.
- Подтверждение исполнения Заказчиком (кто, когда) — неотменяемое, см. [[07. Подтверждение исполнения Заказчиком|Подтверждение исполнения Заказчиком]].
- Подтверждение комплектности документов Исполнителем (кто, когда) — см. [[09. Проверка комплектности и закрытие заявки|Проверка комплектности и закрытие заявки]].
- Начало / окончание / досрочный отзыв [[04. Делегирование полномочий Руководителя|делегирования]] полномочий Руководителя.
- Вход/выход из системы (для расследования инцидентов).

## Требования
- Записи в журнале аудита не редактируются и не удаляются (append-only), см. [[Бизнес-правила|BR-070]].
- Каждая запись: timestamp, пользователь, тип события, ID заявки (если применимо), old_value/new_value (если применимо). Полный минимальный набор аудируемых действий — [[Бизнес-правила|BR-071]].
- Доступ на просмотр журнала — только у Руководителя (и делегированного заместителя на период делегирования), см. [[Матрица прав]].
- Журнал должен попадать в отчёты как минимум по одной заявке (полная история заявки на одном экране/выгрузке) — см. [[Отчётность]].
