# Проверка комплектности и закрытие заявки

Финальный этап [[01. Жизненный цикл заявки|жизненного цикла заявки]]. Наступает после того, как Заказчик [[07. Подтверждение исполнения Заказчиком|подтвердил исполнение]] заявки (статус → `Проверка комплектности документов`, см. [[Статусы заявки]]).

Документы здесь **не загружаются заново** — они уже были загружены Исполнителем по ходу [[00. Исполнение заявки — обзор|исполнения заявки]]. Этот шаг — только проверка комплектности. Дополнительные закрывающие документы (например, полученные позже — в частности, при участии субагента) можно догрузить на любом из под-шагов ниже, вплоть до фактического закрытия.

## Что происходит
1. Система формирует чек-лист обязательных документов для данного вида заявки на основе [[Справочник типов документов]] (в зависимости от того, была это Закупка, Платёж через агента или Платёж через банк).
2. Система автоматически отмечает, какие типы документов уже загружены, и подсвечивает недостающие обязательные.
3. Исполнитель догружает недостающие документы (если условный документ ещё требуется) и нажимает "Отправить на проверку Руководителю" → статус `Проверка комплектности документов` → `На проверке у Руководителя`.
4. Руководитель (или его заместитель с делегированными полномочиями) проверяет комплект документов и принимает решение:
   - **закрыть заявку** → статус → `Закрыта`;
   - **отправить на доработку** — обязательно с комментарием о причине → статус → `Проверка комплектности документов`, заявка возвращается Исполнителю, который догружает недостающее/исправленное и повторно отправляет на проверку (п.3–4 могут повторяться).

## Правила (см. [[Бизнес-правила]])
- **BR-050.** Заявка не может быть закрыта, если отсутствует обязательный документ.
- **BR-051.** Все правила обязательности документов проверяются на сервере, а не только ограничениями интерфейса.
- **BR-052.** Для конкретной заявки может быть создано индивидуальное правило обязательности документа (Руководитель или его заместитель могут переопределить перечень обязательных документов по конкретной заявке, не трогая общий справочник) — такое правило содержит заявку, тип документа, значение обязательности, автора изменения, дату и причину.
- **BR-083.** Заявка закрывается только после: завершения исполнения → подтверждения результата (если требуется) → проверки обязательных документов → проверки Руководителем → выполнения прочих обязательных условий.
- **BR-110.** Заявка закрывается только решением Руководителя (или его заместителя с делегированными полномочиями), после того как Исполнитель отметил комплект документов готовым. Простого подтверждения комплектности Исполнителем недостаточно для закрытия.
- **BR-111.** Руководитель может вернуть заявку на доработку вместо закрытия; отправка на доработку обязательна с комментарием о причине и фиксируется в [[Аудит|аудите]]. Цикл `Проверка комплектности документов ⇄ На проверке у Руководителя` может повторяться неограниченное число раз.

`Закрыта` — терминальный статус, дальнейшие действия по заявке не выполняются, но заявка остаётся видимой в [[Отчётность|отчётах]] и [[Аудит|аудите]].
