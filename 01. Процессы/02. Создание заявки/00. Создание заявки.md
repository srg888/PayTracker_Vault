# Создание заявки

Первый этап [[01. Жизненный цикл заявки|жизненного цикла заявки]]. Выполняется Заказчиком (в текущей версии системы Заказчик и Автор заявки — один и тот же пользователь, см. [[Словарь предметной области]]).

## Правила (см. [[Бизнес-правила]])
- **BR-010.** Заявка создаётся в статусе `Черновик`.
- **BR-011.** До подачи заявки Автор может свободно редактировать её поля.
- **BR-012.** После перехода `Черновик` → `Новая заявка` поля, влияющие на смысл операции (тип заявки, сумма, валюта, получатель, способ платежа, платёжный агент, назначение платежа, срок исполнения), обычным редактированием не изменяются. Для их изменения нужна отдельная процедура.
- **BR-013.** Номер заявки уникален, не строится на основе количества существующих записей (нельзя `COUNT(*) + 1`, см. [[Архитектурные решения|ADR-005]]) и не переиспользуется после удаления заявки.
- **BR-014.** Автор заявки = Заказчик (в текущей версии).

## Типы заявок и их поля
Общие для всех типов поля (заголовок, подразделение/юрлицо заказчика, ожидаемый срок) хранятся в общей сущности `REQUESTS` (см. [[ER модель]]), специфичные — в таблицах-расширениях по типу:

- [[01. Поля заявки — Платёж|Поля заявки — Платёж]]
- [[02. Поля заявки — Закупка|Поля заявки — Закупка]]
- [[03. Поля заявки — Консультация|Поля заявки — Консультация]]

## Проверка перед подачей
Система проверяет, что все обязательные поля заполнены, прежде чем позволить перевести заявку из `Черновик` в `Новая заявка`.
