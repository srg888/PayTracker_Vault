# Согласование условий исполнения (Платёж)

Первый шаг [[00. Исполнение заявки — обзор|исполнения]] **платёжной** заявки — только для типа заявки "Платёж" (для Закупки и Консультации не применяется). Наступает сразу после того, как заявка [[03. Назначение исполнителя|принята Исполнителем]] (статус `В работе`), и предшествует непосредственному исполнению ([[01. Исполнение — Платёж через банк|через банк]] / [[02. Исполнение — Платёж через агента|через агента]]).

## Что происходит
1. Исполнитель выбирает способ платежа — прямой платёж через банк или через платёжного агента ([[Агенты]], если через агента) — и указывает условия: комиссию за платёж и курс, по которому платёж будет выполнен.
2. Исполнитель отправляет предложение Заказчику на согласование → статус заявки `В работе` → `Условия предложены` (см. [[Статусы заявки]]).
3. Заказчик рассматривает предложение:
   - **принимает** → статус возвращается в `В работе`, Исполнитель переходит к фактическому исполнению ([[01. Исполнение — Платёж через банк|Платёж через банк]] / [[02. Исполнение — Платёж через агента|Платёж через агента]]);
   - **отклоняет** — обязательно с комментарием о причине отклонения → статус возвращается в `В работе`, Исполнитель готовит новое предложение (см. п.1).
4. Каждое предложение (способ, комиссия, курс, кто предложил, когда, решение Заказчика, комментарий) сохраняется в истории — предыдущие предложения не удаляются и не перезаписываются при повторном предложении.

## Правила (см. [[Бизнес-правила]])
- **BR-100.** Исполнитель обязан предложить Заказчику способ платежа, комиссию и курс до перехода к фактическому исполнению.
- **BR-101.** Заказчик обязан явно принять или отклонить предложение; при отклонении обязателен комментарий с причиной. История предложений сохраняется целиком (не только последнее).
- **BR-102.** Переход к [[07. Подтверждение исполнения Заказчиком|подтверждению исполнения]] (загрузка платёжки/СВИФТ) возможен только если последнее по времени предложение условий принято Заказчиком.

## Данные
Хранится в отдельной сущности `PAYMENT_TERMS_PROPOSALS` (1:N к платёжной заявке), а не как одно поле в самой заявке — чтобы не терять историю переторговки условий (см. [[ER модель]]). Принятые условия (способ, комиссия, курс) дублируются в саму заявку (`agreed_payment_method`, `agreed_commission_amount`, `agreed_rate`) для быстрого доступа при исполнении и в проверке комплектности.
